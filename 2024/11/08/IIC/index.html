1. I²C通信协议详细介绍

1.1 概述

I²C（Inter-Integrated Circuit），中文称为集成电路总线，是一种由飞利浦公司（现为恩智浦半导体）在20世纪80年代初期开发的串行通信总线。I²C是一种同步、半双工、多主多从的两线制总线，主要用于集成电路之间的短距离通信。

特点：

  • 两线制：仅需两条信号线，即串行数据线（SDA）和串行时钟线（SCL）。
  • 多主多从：允许多个主设备和多个从设备连接在同一总线上。
  • 同步通信：主设备提供时钟信号，同步数据传输。
  • 支持设备寻址：通过设备地址进行通信控制。

1.2 物理层

总线拓扑

I²C总线采用开漏（Open-Drain）或开集极（Open-Collector）驱动方式，需要通过上拉电阻将SDA和SCL线连接到电源正极。总线上所有设备的SDA和SCL引脚都连接在一起。

上拉电阻的作用：

  • 提供逻辑高电平。
  • 限制电流，防止短路。

信号电平

  • 逻辑低电平（0）： SDA或SCL线被任何设备拉低（主设备或从设备）。
  • 逻辑高电平（1）： SDA或SCL线通过上拉电阻拉高（当总线上没有设备拉低时）。

1.3 总线信号

  • SDA（Serial Data Line）： 双向数据线，传输数据位。
  • SCL（Serial Clock Line）： 时钟线，由主设备提供，同步数据传输。

1.4 通信过程

I²C通信由以下基本操作组成：

  1. 起始条件（Start Condition，S）：

    • 当SCL为高电平时，SDA从高电平跳变到低电平，表示通信开始。

  2. 停止条件（Stop Condition，P）：

    • 当SCL为高电平时，SDA从低电平跳变到高电平，表示通信结束。

  3. 数据传输：

    • 数据位在SCL的上升沿或下降沿稳定，并在特定时钟沿采样。
    • 每次传输8位数据（一个字节），高位在前（MSB First）。

  4. 应答位（Acknowledge，ACK）：

    • 发送方发送8位数据后，第9个时钟周期由接收方拉低SDA线，表示应答（ACK=0）。
    • 如果接收方不拉低SDA线，则表示非应答（NACK=1）。

1.5 设备寻址

I²C设备地址通常为7位或10位，常用的是7位地址。

  • 7位地址：

    • 高7位为设备地址，第8位为读写位（R/W bit）。
    • R/W位：0表示写操作，1表示读操作。

1.6 读取与写入操作

写入操作

  1. 起始条件（S）
  2. 设备地址 + 写位（0）
  3. 从设备应答（ACK）
  4. 寄存器地址或子地址（可选）
  5. 从设备应答（ACK）
  6. 数据字节
  7. 从设备应答（ACK）
  8. 停止条件（P）

读取操作

  1. 起始条件（S）
  2. 设备地址 + 写位（0）
  3. 从设备应答（ACK）
  4. 寄存器地址或子地址（可选）
  5. 从设备应答（ACK）
  6. 重复起始条件（Sr）
  7. 设备地址 + 读位（1）
  8. 从设备应答（ACK）
  9. 数据字节
  10. 主机非应答（NACK）
  11. 停止条件（P）

1.8 高级特性

时钟伸展（Clock Stretching）

  • 从设备在需要延迟时，可拉低SCL线，迫使主设备等待，从而延长时钟周期。

多主控制

  • 总线上允许多个主设备，但在同一时间只能有一个主设备主动通信。

  • 仲裁机制：

    • 当多个主设备同时发起通信时，检测到SDA线被拉低，未拉低的主设备停止通信。

2. RTOS下的I²C驱动框架和实例

2.1 RTOS下的驱动设计原则

  • 实时性： 确保驱动程序满足实时性要求，及时响应I²C通信事件。
  • 可移植性： 尽可能编写与硬件无关的代码，便于在不同平台上移植。
  • 模块化： 将驱动程序分为硬件层、接口层和应用层。
  • 资源管理： 合理使用RTOS的资源，如任务、队列、信号量等。

2.2 I²C驱动框架

1. 硬件抽象层（HAL）

  • 提供对底层硬件寄存器的访问和控制。
  • 实现基本的I²C操作，如发送起始条件、发送停止条件、读写数据位等。

2. 驱动接口层

  • 基于HAL，提供上层应用调用的API函数。
  • 实现I²C初始化、读写操作、错误处理等功能。

3. 应用层

  • 调用驱动接口层提供的API，实现具体的传感器或设备的通信。

3. Linux下的I²C驱动框架和实例

3.1 Linux I²C子系统概述

Linux内核提供了完善的I²C子系统，包括I²C 适配器驱动和I²C 设备驱动。

  • I²C适配器（Adapter）： 对应I²C控制器驱动，负责底层硬件的操作。
  • I²C设备（Device）： 对应连接在I²C总线上的从设备驱动。

I²C子系统主要组成：

  • I²C核心层（i2c-core）： 管理I²C总线、适配器和设备的注册、注销和调度。
  • I²C算法层（i2c-algo）： 定义I²C操作的算法，如位操作、SMBus协议支持等。
  • I²C适配器层（i2c-adapter）： 实现与具体硬件控制器相关的代码。
  • I²C设备层（i2c-client）： 实现具体I²C从设备的驱动代码。

3.2 I²C总线驱动与适配器
